30 Interactive HTML Pages

This page collects the standalone interactive HTML pages for the book. Each page is designed as a short visual activity that supports the corresponding chapter. Students can use these pages before class for intuition, during class for guided exploration, or after class for review.

The pages live in the htmls/ folder of the repository. When the book is published through GitHub Pages, the links below open the corresponding interactive page directly from the book site. When the book is rendered locally, the same links open the local files in the htmls/ folder.

30.1 How to use these pages

A good workflow is:

  1. Read the chapter section that introduces the concept.
  2. Open the corresponding interactive page.
  3. Change parameters, rotate figures, or compare examples.
  4. Return to the chapter and solve the worked examples or exercises.
  5. Use the Python lab for a deeper computational version.

For instructors, these pages can also be used as short classroom demonstrations. The best use is not to show a finished picture, but to ask students what should happen when a parameter changes and then test the prediction.

30.3 Suggested development plan

The current htmls/ folder can be developed in stages.

Stage 1: Concept visualization. Each page should include a simple interactive figure: a slider, toggle, contour plot, vector field, or rotating 3D surface.

Stage 2: Guided questions. Each page should include two or three short prompts, such as “What changes?”, “What stays fixed?”, and “What does the formula predict?”

Stage 3: Computation bridge. Each page should link naturally to the corresponding Python lab, where students reproduce or extend the visualization with code.

Stage 4: Assessment. Each page can end with a short check-for-understanding question that can be used in class, discussion sections, or review assignments.

30.4 Design principles

The interactive pages should follow the same philosophy as the book:

  • start from geometry and visual intuition,
  • connect formulas to computation,
  • use examples from data, probability, optimization, and applied mathematics,
  • keep each activity short enough to use in a focused class moment,
  • help students move from “I can see it” to “I can compute it” to “I can explain it.”

30.5 File naming convention

The HTML files use a simple chapter-based convention:

htmls/chapter-01.html
htmls/chapter-02.html
...
htmls/chapter-27.html

This convention keeps the links stable even if chapter titles are revised later.
