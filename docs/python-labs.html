29 Python Labs

This page collects the Python labs for the book. Each lab is a Jupyter notebook designed for computation, visualization, simulation, and modeling. The labs can be opened directly in Google Colab, so students can run the code in a browser without installing Python first.

29.1 How to use the labs

  1. Click Open in Google Colab for the chapter you are studying.
  2. In Colab, choose Runtime > Run all to execute the notebook from top to bottom.
  3. To edit and save your own version, choose File > Save a copy in Drive.
  4. For local work, download or clone the repository and open the notebook from the labs/ folder.

The lab filenames are chapter-based and do not use course-number prefixes.

29.3 Suggested workflow for instructors

The labs are intended to support a modern, computation-rich multivariable calculus course. A typical use pattern is:

  • before class: students run selected visualization cells to prepare for geometric concepts;
  • during class: the instructor modifies parameters live to show how formulas respond to changes;
  • after class: students complete short computational exercises and submit selected plots, explanations, or numerical conclusions;
  • before exams: students revisit the labs as concept maps connecting formulas, geometry, and computation.

29.4 Suggested workflow for students

Do not treat the labs as code to copy mechanically. For each notebook, focus on three questions:

  1. What mathematical object is being computed or visualized?
  2. Which formula or theorem explains the computation?
  3. How does the output change when the parameters, region, vector field, or model are changed?

A good lab write-up should combine code, plots, formulas, and short mathematical explanations.
